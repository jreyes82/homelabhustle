Network+ Study Notes: Subnetting — Why We Need It and How to Do It

Study Notes April 2026 5 min read

Subnetting was not new to me going into Network+. I had worked with IP addresses throughout my IT career — assigning them, troubleshooting connectivity, configuring basic network gear. But I had never gone deep on the math behind it. I knew how to use subnets. I did not really understand how they worked. The Network+ curriculum changed that, and this post covers everything I learned going further than I ever had before.

Why Subnetting Exists

Before IPv6 came along, the internet ran on IPv4. IPv4 uses 32-bit addresses which gives you a maximum of about 4.3 billion unique addresses. That sounds like a lot, but the early designers of the internet made some decisions about how to allocate those addresses that turned out to be very wasteful.

Originally, IP addresses were divided into classes:

  • Class A — first octet 1-126. Supports 16 million hosts per network. There are only 126 Class A networks in existence.
  • Class B — first octet 128-191. Supports 65,534 hosts per network.
  • Class C — first octet 192-223. Supports 254 hosts per network.

The problem with classful addressing is the waste. If a company needed 300 hosts, they would get a Class B network which supports 65,534 hosts. That is 65,234 addresses sitting unused, permanently unavailable to anyone else. Multiply that across thousands of organizations and you burn through your address space fast.

Subnetting was the solution. Instead of handing out massive fixed blocks, subnetting lets you take a network and carve it up into smaller, more appropriately sized pieces. A company that needs 300 hosts gets a subnet that supports around 510 — not 65,534. The rest of that address space goes back into the pool.

Exam tip: Know the three IP address classes, their ranges, and default subnet masks cold. Class A = /8, Class B = /16, Class C = /24. These show up constantly.

Understanding IP Addresses in Binary

You cannot truly understand subnetting without understanding binary. The good news is you only need to understand one concept: an IP address is 32 bits, divided into four groups of 8 bits called octets. Each octet can be a value from 0 to 255.

Each bit position in an octet has a value:

128  64  32  16   8   4   2   1
  1   1   0   0   0   0   0   0  =  192
  1   0   1   0   1   0   1   0  =  170
  1   1   1   1   1   1   1   1  =  255
  0   0   0   0   0   0   0   0  =  0

To convert an octet to binary, work from left to right. Does 128 fit? Yes — write a 1 and subtract. Does 64 fit in the remainder? And so on.

Exam tip: Memorize these eight values: 128, 64, 32, 16, 8, 4, 2, 1. You will use them constantly during the exam. Also memorize the common sums: 192 (128+64), 224 (128+64+32), 240 (128+64+32+16), 248, 252, 254, 255.

What a Subnet Mask Does

A subnet mask is also a 32-bit number, written alongside an IP address. Its job is to define which part of an IP address is the network portion and which part is the host portion.

In a subnet mask, all the 1 bits represent the network portion, and all the 0 bits represent the host portion. The 1s always come first, followed by the 0s — they never alternate.

IP Address:    192.168.1.100
Subnet Mask:   255.255.255.0

In binary:
IP:   11000000.10101000.00000001.01100100
Mask: 11111111.11111111.11111111.00000000
      |-------- network --------|--host--|

In this example the first three octets (24 bits) are the network, and the last octet (8 bits) is the host. This is a /24 network — which brings us to CIDR.

CIDR Notation

CIDR stands for Classless Inter-Domain Routing. It is a shorthand way of writing a subnet mask. Instead of writing 255.255.255.0, you write /24 — because there are 24 ones in the subnet mask.

255.0.0.0       = /8
255.255.0.0     = /16
255.255.255.0   = /24
255.255.255.128 = /25
255.255.255.192 = /26
255.255.255.224 = /27
255.255.255.240 = /28
255.255.255.248 = /29
255.255.255.252 = /30
Exam tip: Memorize the entire table above. Know the CIDR prefix, the dotted decimal subnet mask, the number of hosts per subnet, and the block size. The exam will test all of these.

Calculating the Number of Hosts

Once you know the subnet mask, calculating the number of usable hosts is straightforward:

Usable hosts = 2^(host bits) - 2

You subtract 2 because the first address in every subnet is the network address and the last address is the broadcast address. Neither can be assigned to a device.

/24 = 8 host bits  = 2^8  - 2 = 254 usable hosts
/25 = 7 host bits  = 2^7  - 2 = 126 usable hosts
/26 = 6 host bits  = 2^6  - 2 = 62 usable hosts
/27 = 5 host bits  = 2^5  - 2 = 30 usable hosts
/28 = 4 host bits  = 2^4  - 2 = 14 usable hosts
/29 = 3 host bits  = 2^3  - 2 = 6 usable hosts
/30 = 2 host bits  = 2^2  - 2 = 2 usable hosts
Exam tip: /30 networks (2 usable hosts) are used almost exclusively for point-to-point links between routers. If you see /30 on the exam, think router-to-router connection.

Finding the Network Address

The network address is the first address in a subnet. It identifies the subnet itself and cannot be assigned to a host. To find it, perform a bitwise AND between the IP address and the subnet mask.

Example: What is the network address for 192.168.1.130/26?

Subnet mask for /26:
11111111.11111111.11111111.11000000 = 255.255.255.192

IP in binary:   11000000.10101000.00000001.10000010
Mask in binary: 11111111.11111111.11111111.11000000
AND result:     11000000.10101000.00000001.10000000

Network address = 192.168.1.128

The shortcut: for /26 the block size is 64 (2^6 = 64). Subnets fall on multiples of 64: 0, 64, 128, 192. The address 130 falls between 128 and 192, so the network address is 192.168.1.128.

Exam tip: Learn the block size shortcut — it is much faster than binary AND during the exam. Block size = 256 minus the last non-255 octet of the subnet mask. For 255.255.255.192, block size = 256 - 192 = 64.

Finding the Broadcast Address

The broadcast address is the last address in a subnet. It is always one less than the next network address.

Network address      = 192.168.1.128
Block size           = 64
Next network address = 192.168.1.128 + 64 = 192.168.1.192
Broadcast address    = 192.168.1.192 - 1  = 192.168.1.191

Finding the Host Range

The usable host range is everything between the network address and the broadcast address:

Network address:   192.168.1.128
First usable host: 192.168.1.129
Last usable host:  192.168.1.190
Broadcast address: 192.168.1.191

Putting It All Together - A Full Example

Given: 172.16.45.200/20 — find the network address, broadcast address, and host range.

Step 1 - Subnet mask for /20
255.255.240.0

Step 2 - Block size
Interesting octet is the third: 256 - 240 = 16
Subnets in third octet: 0, 16, 32, 48...

Step 3 - Which subnet does 45 fall in?
45 falls between 32 and 48
Network address = 172.16.32.0

Step 4 - Broadcast address
Next network = 172.16.48.0
Broadcast    = 172.16.47.255

Step 5 - Host range
First host: 172.16.32.1
Last host:  172.16.47.254

Step 6 - Usable hosts
/20 = 12 host bits = 2^12 - 2 = 4094 usable hosts
Exam tip: When the interesting octet is not the last one, all bits after it are host bits. For /20, the third octet has 4 host bits and the entire fourth octet adds 8 more — 12 host bits total.

Number of Subnets

The exam may ask how many subnets a given mask creates within a classful network. The formula is:

Number of subnets = 2^(subnet bits borrowed)

Starting with a Class C network (/24) and subnetting to /26 means borrowing 2 bits:

2^2 = 4 subnets
Each /26 has 62 usable hosts
4 x 64 = 256 -- the full /24 address space accounted for

Private IP Address Ranges

These ranges are reserved for private use and are not routed on the public internet:

Class A: 10.0.0.0    - 10.255.255.255   (/8)
Class B: 172.16.0.0  - 172.31.255.255   (/12)
Class C: 192.168.0.0 - 192.168.255.255  (/16)
Exam tip: Any IP in these ranges is private and requires NAT to reach the public internet. Know these ranges without hesitation.

Special Addresses to Know

  • 127.0.0.1 — loopback. Tests the local network stack. Traffic never leaves the machine.
  • 169.254.x.x — APIPA. Assigned automatically when a device cannot reach a DHCP server. If you see this on a client, DHCP is not working.
  • 0.0.0.0 — unspecified address. Used before a device has an IP assigned.
  • 255.255.255.255 — limited broadcast. Sent to all hosts on the local segment.
Exam tip: 169.254.x.x is a major hint on the exam. Any scenario where a host cannot communicate and has a 169.254 address means the DHCP server is unreachable.

Quick Reference Cheat Sheet

Exam Strategy

The Network+ exam will give you subnetting questions under time pressure. Here is the approach that works:

  1. Identify the CIDR prefix and convert to subnet mask if needed
  2. Find the interesting octet — the one that is not 255 or 0
  3. Calculate the block size: 256 minus the interesting octet value
  4. Find which block the IP address falls in
  5. Network address = start of that block
  6. Broadcast = start of next block minus 1
  7. Host range = network + 1 to broadcast - 1

Practice this process until you can do it in under 60 seconds. Use real IP addresses and work through at least 20 problems before the exam. The math is always the same — once the steps are automatic, subnetting questions become some of the easiest points on the test.

I already knew subnetting existed and had used it in the field. What the Network+ material gave me was the understanding of why each step works the way it does. Going deeper than surface level made the whole thing click in a way that just configuring gear never did.
